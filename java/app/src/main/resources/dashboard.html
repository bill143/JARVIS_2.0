Pick a chat brain by adding an API key. Anthropic is native; every other option is an OpenAI-compatible endpoint (NVIDIA, OpenAI, OpenRouter, Groq, Mistral, DeepSeek, xAI, or a local Ollama). Keys stay on this PC and are never shown back. Test a key before you trust it; Activate switches the brain live.
Add as many providers as you like — each with its own key. Pick one as the active brain; switching applies live, no restart.

🔗 MCP connections

Connect JARVIS to Model Context Protocol servers over HTTP. Once connected, the server's tools become available to JARVIS (governed and audited like any other tool). Dormant until you add one; connections are stored locally.
No MCP servers configured.

🧩 Connectors

Configure the external connectors in-app — no environment variables, no restart. Saved values are stored locally and take effect on the next request. A saved value overrides its environment variable; clearing a field reverts to the environment. Secrets are never shown back.

🛡 Gated local lane (advanced)

A policy-gated route to a model you self-host (e.g. a local Ollama) for domain tasks a cloud provider declines on scope grounds. Off by default; default-deny. A task runs only if the lane is enabled, points at a self-hosted endpoint, matches your allowlist, and does not hit the absolute harm denylist (which always blocks). Every gate decision and run is audited; lane output is flagged for review.

⚡ Team 2.0 — dynamic roles · parallel · self-correcting · budgeted · human-in-the-loop

The orchestrator picks only the roles your task needs, runs independent sub-tasks in parallel, grounds the shared scratchpad on your unified knowledge (Brain/Memory/Intelligence), and lets the Critic send feedback back to the Executor. Set optional budgets; you can redirect the team mid-run.
